## C언어 개요

- 컴퓨터와 가장 가까운 언어
- C언어는 하드웨어와 소통할수있을정도로 저수준에 가깝기 때문에 컴퓨터의 구조와 동작원리의 깊이있게 배울 수 있음
- 다른 프로그래밍 언어의 밑바탕이 되는 언어
- 메모리 효율성을 극대화할수있는 언어
- 메모리의 모든관리 직접 가능, 필요한 만큼 쓰고 버릴수있음
- 성능이 중요한 프로그램에서 매우 중요(임베디드, 운영체제, 게임등)
- 컴파일 : 우리가 작성한 프로그렘을 컴퓨터가 이해할수있는 기계어로 번역하는것
- 파이썬과는 다르게 자동으로 개행되지 않음 ← 제어문자 활용

## 구조

- #include
    - 지정한 파일의 내용을 읽어오기 위한 명령어
    - stdio.h라는 헤더파일을 읽어오는것
    - 표준 입출력 함수가 저장되어있다
- main 함수
    - C프로그램은 함수로 만들어지며, 함수는 일정기능을 수행하는 단위
    - 프로그램의 시작을 의미, 반드시 있어야함
    - return 0; 명령어는 프로그램이 성공적으로 끝났다는 신호를 OS에 전달하고, 제어 흐름을 명확하게 종료하기 위함
    
- 세미콜론
    
    각 문장의 끝에는 종료지점이라는 의미의 세미콜론을 ***반드시*** 찍어야함
    
- printf() 함수
    - 문자열 출력 함수
- 주석
    - 개요
        - 소스 코드를 설명하는 주석또는 남겨두고 싶은 메모 작성
        - 주석 처리된 문장은 프로그램 실행할때 영향 X
        - 여러 명이 함께 코드를 작성하는 경우 필수 작업
    - 사용빙법
        - //[]// 부터 해당줄의 끝까지 주석 처리
        - /*~*/ [] /*~*/ 사이의 모든 내용을 주석 처리
    - 단축키
        - 주석 처리 command + K + C
        - 주석 해제 command + K + U

## 변수

- 데이터를 저장하기 위한 공간
- 변수 선언을 통해 실제 메모리에 저장 공간을 확보하여 사용
- 어떤 값을 저장할 것인가에 따라 자료형을 다르게
- 변수 선언
    - 유의 사항
        - 변수명은 용도에 맞게 만들어 주는것이 좋음(가독성 향상)
        - 예약어와 식별자
            - 예약어 : 컴파일러와 사용 방법이 약속된 단어
            - 식별자 : 필요에 따라 만들어 사용하는 단어
            - int age; int는 에약어, age는 식별자
        - 식별자 생성 시 규칙
            - 알파벳 대문자, 소문자, 숫자, _(밑줄)로 만들 수 있음
            - 숫자로 시작할수없음
            - 대문자와 소문자는 서로 다른 식별자로 인식
            - 예약어는 식별자로 사용할 수 없음
    
    ```c
    #include <stdio.h>
    int main(void){
    	int a;
    	char ab;
    	float d;
    }
    ```
    
- 등호는 오른쪽의 값을 왼쪽에 넣는 대입 연산자
- 변수를 선언하고 아무 값도 저장하지 않는다면 쓰레기값이 저장됨

## 자료형

- 중괄호 내에 변수를 선언 시 중괄호 내에서 어디든지 사용 가능( 단, 선언 전 부분에 사용은 불가능)
- 변수 선언 시 한 줄로 여러 개 작성 가능(int a, b, c;의 형태로도 가능, 단,서로 다른 자료형은 세미콜론으로 구분되어야함)
- 등호를 기준으로 왼쪽은 공간, 오른쪽은 값
- 숫자 변수
    - 정수형 실수 : 정수를 저장하기 위한 메모리 공간
    - 실수형 변수 : 실수를 저장하기 위한 메모리 공간
- 문자 변수
    - 문자형 변수 : 문자를 저장하기 위한 메모리 공간(작은 따옴표)
    - 문자열 : C언어에서는 문자열을 저장하기위한 변수는 없음(큰 따옴표) 대신 배열을 활용
- 종류
    
    ```c
    정수형
    - int : 1byte 
    - long long : 8byte
    
    실수형
    -float : 4byte
    -double : 8byte
    
    문자형
    -char : 1byte
    
    논리형
    -bool : 1byte
    ```
    
- C언어에서 문자는 아스키 코드로 이루어짐
    
    
    | 10진 | 문자 |

## 입력 함수

- stdio.h는 표준입출력(standard input& output), 따라서 입력하는 함수도 존재
- scanf() 함수는 C언어에서 가장 대표적인 입력 함수(키보드를 통해 입력)
- (파이선의 input() 함수와 동일, 단 서식 지정자에 의해 입력 받은 값의 자료형 결정)
- 기본 원형은 scanf(”서식 지정자”, &변수);
- &의 역할
    - 주로 연산자로 변수의 주소를 가져옴
    - printf() 함수는 변수 안에 있는 값을 들고 오기만 하면 되지만
    - scanf() 함수는 변수 안에 값을 써야 하기 때문에 주소에 접근해야함
    - 각 값을 입력할때는 ‘엔터’, ‘스페이스바’, ‘탭’으로 값을 구분해야함

## 연산자

- 연산자 종류

```c
대입 : =                      비트 : &, |, ^, <<, >>

산술 : +, -, *, /, %          증감 : ++,--

비교 : >, >=, <, <=, == ,!=   복합 대입 : +=, -=, *=, /=, %=, &= ..등등

논리 : &&, ||, !              참조 : &, *
```

- 비교 연산자(관계 연산자)
    - 두 개의 피연산자 간 대소 관계 또는 동등 관계를 확인하여 명제 결과로 출력
    - 연산의 결과값은 1(True) 또는 0(False)
    - 컴퓨터는 0은 거짓, 0이 아닌 값은 참으로 판단, 단 1을 대표로 사용
- 논리 연산자
    - 하나 이상의 명제를 비교
    - 결과값은 1(True) 또는 0(False)
- 그 외 연산자
    - 형 변환 연산자
        - 피연산자 값의 자료형을 원하눈 형태로 변환 시킴
        - 단, 변수에 저장된 값이 변하지는 않음
        
        ```c
        int a = 10;
        printf("%f", (float)a);
        ```
        
    - 복합 대입 연산자
        - 대입 연산자와 다른 연산자(산술, 비트)를 하나로 합친 것
    - 콤마 연산자
        - 한번에 여러 개의 수식을 차례로 나열 해야 할 때 사용
        - 왼쪽부터 오른쪽을 차례로 연산을 수행, 가장 오른쪽의 피연산자가 최종 결과값
    - 조건 연산자
        - 조건의 결과에 따라 결과값이 달라지느 연산자
        - 코드를 간략하게 만들거나 메크로 함수에 주로 사용
    - 비트 연산자
        - 데이터를 비트 단위로 연산
        
        | 기호 | 이름 | 의미 | 예시 |
        | --- | --- | --- | --- |
        | & | AND | 두 비트가 모두 1이면 1 | 6 & 3 → 110 & 011 = 010(2) |
        | | | OR | 두 비트 중 하나라도 1이면 1 | 6 | 3 → 110 | 011 =. 117 |
        | ^ | XOR | 두 비트가 서로 다른면 1 | 6 ^ 3 → 110 ^ 011 =2.853×10²² 105(5) |
        | ~ | NOT | 비트를 반전(0 → 1, 1→ 0) | ~6 → 11111001(-7) |
        | << | 왼쪽 시프트 | 비트를 왼쪽으로 n칸 이동 | 3 << 1 → 011 → 110(6) |
        | >> | 오른쪽 시프트 | 비트를 오른쪽으로 n칸 이동 | 6 >> 1 → 110 → 011(3) |

## 조건문

- 조건문 작성 시 논리 연산자를 함께 사용하면 더 다양한 조건식을 작성할 수 있음
- if 문
    - 조건의 결과에 따라 시행되는 명령문을 결정하는 구조
    - 조건을 만족할 떄 실행할 문장이 한 줄이면 중괄호 생략 가능
    - (두 줄 이상이면 무조건 중괄호가 있어야 함)
    
    ```c
    #include <stdio.h>
    int main(void){
    	int a = 20, b = 10;
    	if(a>b){
    		printf("a>b\n"); // 참일 때 수행될 명령어;
    		}	
    }
    ```
    
- if - else문
    - 조건이 참일 때와 거짓일 때 각각 수행할 명령어를 작성하여 수행
    - else문에는 조건을 붙일 수 없음
    - if문과 마찬가지로 실행문이 한 줄이면 중괄호 생략 가능
    - 실행문 중 하나를 무조건 선택해야 하는 경우는 if else문 사용권장
    
    ```c
    #include <stdio.h>
    int main(void){
    	int a = 20, b = 10;
    	if(a>b){
    			printf("a > b\n"); // 참일 때 수행될 명령어;
    		}	
    	else{
    			printf("a < b\n"); // 거짓일 때 수행될 명령어;
    	}
    }
    ```
    
- if -else if -else 문
    - 3개 이상의 실행문 중 하나를 선택하느 경우(조건이 2개 이상인 경우)
    - 연산 효율성 up !
    
    ```c
    #include <stdio.h>
    int main(void){
    	int a = 20, b = 10;
    	if(a>b){
    			printf("a>b\n"); // 참일 때 수행될 명령어;
    		}
    	else if(a < b){
    			printf("a < b\n"); // 거짓일 때 수행될 명령어;
    		}
    	else{
    			printf("a == b\n");
    	}
    }
    ```
    
- switch-case 문
    - 조건의 결과에 따라 시행되는 명령문을 결정하는 구조
    
    ```c
    #include <stdio.h>
    
    int main(void){
    		char ch;
    		printf("영어 소문자를 입력하세요 : ");
    		scanf("%c", &ch);
    		switch (ch){
    				case 'a' : printf("모음"); break;
    				case 'e' : printf("모음"); break;
    				case 'i' : printf("모음"); break;
    				case 'o' : printf("모음"); break;
    				case 'u' : printf("모음"); break;
    				default : printf("자음");
    		}
    }
    ```
    
    | case 값 |   • 조건의 결과가 값 n일때 해당 명령 실행
      • 값 n은 반드시 정수형 상수, 변수나 실수는 올 수 없음 |
    | --- | --- |
    | break ; |   • 현재 진행중인 선택 구조 or 반복 구조를 곧바로 벗어나는 연산자
      • switch-case 함수에는 break가 없으면 다른 명령어도 실행됨 |
    | default |   • case에 없는 그 외 모든경우에 실행되는 명령문(else문)
      • default는 마지막에 실행되기 때문에 굳이 break; 할 필요는 없음 |
    | if 와 차이점 |   • if 함수는 위에서 아래로 내려가면서 모든 조건문을 확인하는 방식
      • switch-case 함수는 해당하는 조건으로 곧바로 분기 |

## 반복 구조

- while
    - 조건식을 먼저 검사하고 조건식이 참인동안 실행문 반복
    - while 함수에서도 실행문이 하나라면 중괄호 생략 가능
    
    ```c
    while(조건식){
    실행문 1;
    실행문 2;
    ...;
    }
    ```
    
    ```c
    #include <stdio.h>
    int main(void){
    		int a = 1;
    		
    		while(a < 10){
    				a = a * 2;
    				printf("a : %d\n", a);
    		}
    		
    		return 0;
    		
    }
    ```
    
- do-while
    - 우선 반복할 문장을 한 번 수행한 후에 조건을 검사, 조건을 만족하면 계속 반복
    
    ```c
    do{
    	실행문 1;
    	실행문 2;
    	...;
    }
    while(조건식);
    ```
    
    ```c
    #include <stdio.h>
    int main(void){
    		int a = 1;
    		
    		do{
    				a = a * 2;
    				printf("a : %d\n", a);
    			}while(a < 10);
    			
    			return 0;
    }
    ```
    
- for
    - 조건이 만족하는 동안 실행문을 반복(원하는 횟수 만큼 반복할 때 자주 사용)
    - 실행문이 하나라면 중괄호 생략 가능
    - 반복 횟수를 제어하기 위해 일반적을 i, j, k를 사용
    - 반복 횟수가 정해진 경우 일반적으로 for 함수 사용(직관적)
    
    ```c
    for 함수 주의점
    
    * 초기식, 조건식, 증감식은 반복 횟수를 알기 쉽게 작성
    for(i = 0; i < 3; i++) //i의 값이 0, 1, 2일때 세 번 반복
    
    * 반복 횟수를 세는 변수를 반복문 안에서 바꾸지말것
    for(i = 0; i < 10; i++)
    {
    		printf("Be happy!\n");
    		i += 2;
    ```

## 배열과 문자열

- 배열
    - 개요
        - 같은 자료혀의 데이터를 저장할 수 있는 **연속된 메모리 공간**
        - 파이선의 listdhk 동일한 형태이지만 “동일한 자료형만” 저장할 수 있다는 점이 큰 차이
        - 배열의 각 방 번호를 ‘인덱스’라고 하며, 0번 인덱스부터 시작
        - 반드시 크기를 사전에 선언해야함(동적 할당, 크기 생략 후 초기화는 예외)
        - 배열의 인덱스는 0붙터 시작해서 크기 -1까지
    - 선언 방법
        - 자료형 배열이름[크기]; - 크기는 값의 개수-
        - 자료형 배열 이름[] = {값1, 값2,…, 값 n} -n개 크기로 배열 선언-
    - 반복 구조
        - 배열은 일반적으로 많은 데이터를 취급하기 때문에 하나하나 접근하면 비효율적 따라서 반복 구조와 함께 사용하는 경우가 많음
        
        ```c
        #include <stdio.h>
        int main(void){
        		int score[10] = {95, 100, 98, 78, 65, 55, 78, 52, 97, 98};
        		int i;

        				printf("%d ", score[i]);
        		}
        }
        ```
        
    - 초기화
        - 배열도 변수와 동일하게 초기화하지 않으면 쓰레기 값이 저장됨
    
- 문자열
    - 개요
        - 문자를 나열하여 저장하기 위해 char형으로 선언된 배열로, null 문자로 끝남
        - C언어는 “문자열”이라는 개념이 없기 때문에 null 문자로 구분
    - %s 서식 지정자
        - 문자열의 출력과 입력에 사용되는 서식 지정자
        - 주소 값을 활용하기 때문에 printf, scanf 각각에서 str(배열 이름은 주소 값)만 사용
        - printf(”%s”, str) - 문자열 배열 str에서 null을 만날때까지 문자 출력
        - scanf(”%s”, str) - 문자형 배열 str에 문자열 입력(마지막 null은 자동 추가)
        - 이때 남아있던 내용은 그대로 저장되어 있음
        - **따라서 char 형 배열 선언 시 배열크기는 최소한 ‘문자열의 길이 + 1’**
    - strcpy
        - 문자열을 대입하는 함수
        - strcpy(저장될 배열명, 저장할 문자열(또는 배열명))
        - string.h - 문자열 관련 함수 원형을 모아 놓은 헤더 파일
    - scanf 의 동작(버퍼 이해하기)
        
        ```c
        * scanf() 함수

        단, 공백, 탭, 개행을 같이 읽어오지 않기 때문에 문제가 생길 수 있음
        
        해결 방법 1
        - %앞에 공백 추가
        ```

## 함수

- 개요
    - printf(), scanf() 처럼 어떤 기능을 수행하는 코드 단위
    - 이미 정해진 함수를 ‘ 표준 라이브러리 함수 ‘

- 함수를 만들고 동작하는 과정
    - 함수 정의 : 함수를 실제 코드로 만드는 것(기능 구현)
    - 함수 호출 : 함수를 사용하기 위해 호출하는 것(printf() << printf() 함수 호출)
    - 함수 선언 : 어떤 함수를 사용할 것 인지 컴파일러에게 정보를 주는 역할(stdio.h)

    - 함수명 : 함수의 이름을 무엇으로 할 것 인지
    - 매개 변수 : 함수가 기능을 수행하는데 필요한 데이터는 무엇인지
    - 반환형 : 함수가 수행된 결과가 무엇인지(반환될 값의 자료형)
- 함수 원형
    
    ```c
    반환형 함수명(매개변수 1, 매개변수 2)
    {
    		//함수가 수행하는 명령, 결괏값을 돌려보냄
    }
    ```
    
- 매개변수 & 반환값이 없는 함수

        - 반환할 값에 따라 함수의 자료형을 작성(int, long long , float, char, double)

    - 외부 소스에서 접근 불가능

## 포인터와 배열

- 개요
    - 변수 공간에 값을 저장 하는 것이 아닌 메모리 주소를 저장하여 해당 공간에 접근하는, 즉 특정 공간 주소를 가리키는 변수(데이터 타입)
- 사용하는 이유
    - 함수에서 여러 값을 반환해야 하는 경우
    - 함수에서 변수를 실제로 수정해야 할 때
    - 동적 메모리 할당
    - 배열과 문자열 처리
    - 복잡한 자료 구조를 만들때 (연결 리스트, 트리 등)
- 메모리
    - 데이터를 넣고 꺼내쓰는 공간, 바이트 단위로 구분됨
    - 변수의 주소 값은 그 변수가 할당된 메모리 공간의 시작 주소
    - %p는 메모리 주소를 16진수 형태로 출력하는 서식 지정자
    - &는 주소 연산자
- 선언 방법
    - 변수 앞에 *만 붙이면 됨
    - **단, 자료형은 주소 위치에 있는 변수의 자료형을 적어야함**
    - ex) int *ptr; 정수 값이 존재하는 메모리 주소를 저장할 수 있는 변수 ptr 선언
    - * 연산자의 역할은 2가지임
    - int *ptr에서 *는 포인터 변수를 생성한다는 뜻이고, *ptr = 10; 에서 *는 역참조 연산자로 사용(주소를 통해 값에 접근하는 것)
    - 포인터에는 항상 특정 주소 값이나 NULL로 초기화
    - 초기화하지 않으면 쓰레기 값(알수없는 주소 공간) 저장
- 문제점
    - 전역 변수로 하면 안되나?
    - 전역 변수 이름 변경 시 해당 변수를 사용하는 모든 곳의 이름도 변경해야함
    - 코드 블록 내 같은 이름의 지역 변수 선언시 해당 전역 변수 사용 불가
    - 전역 변수의 값이 잘못된 경우 접근 가능한 모든 함수를 확인해야함
    - 따라서 통제 가능한 지역 변수를 우선적으로 사용하는 것이 좋음!
- 세부 개념
    - 주소와 포인터의 차이
        - 주소는 프로그램 실행 중에는 그 값이 바뀌지 않는 ‘상수’
        - 포인터는 어떤 주소를 대입 하는 지에 따라 값이 바뀌는 ‘변수’
        - 따라서 서로 다른 포인터 변수가 같은 주소를 공유하는 것이 가능 하지만 주소는 상수이기 때문에 주소에 대입하는 것은 불가능
    - 포인터 변수의 크기
        - sizeof() 연산자를 활용하여 확인 할 수 있음
        - 컴파일 환경에 따라 달라짐
    - 포인터 변수의 자료형은 왜 가리키는 변수의 자료형으로 해야 할까
        - 변수의 크기를 가늠하기 위함, 배열과도 연관
        - 역참조 연산한 값을 어떻게 읽을 것 인지 결정하기 위함
    - 포인터 변수에 연산이 가능할까?
        - 포인터 변수도 변수이기 때문에 연산 가능(산술 연산, 관계 연산)
        - 단 산술 연산은 덧셈, 뺄셈, 증감 연산만 가능
        - 이때 일반 변수와 다른점은 덧셈, 뺄셈읜크기 1이 자료형의 크기로 변환됨
        - int형 포인터 변수에 1은 4만큼, double형 포인터 변수에 1은 8만큼, char형 포인터 변수에 1은 1만큼 더해지거나 빼짐
- 배열과 포인터
    - 배열
        - 동일한 자료형을 저장 할 수 있는 연속된 메모리 공간!!
        - 예를 들어, int ary[5]; 라고 배열이 선언되면 배열의 주소는 아래처럼 생김
        - 정수형은 4byte씩 구성 & 배열은 연속된 메모리 공간 & 주소는 바이트 단위
    - 배열의 첫번째 요소를 활용
        - int *ptr = &ary[0]; → ary[0]번지 주소인 100이 ptr에 저장
        - 이때 배열명은 배열의 시작 주소를 뜻하기 때문에 int *ptr = ary;로 저장 할 수도 있음
        - 주소 접근 : 주소 값에 1씩 더해주면 됨
    - 유사점
        - 배열은 사용하기 조금 편리하게 만든 포인터
        - 단, 배열명과 포인터가 서로 동일한 것은 아님
        - sizeof()의 크기가 다름
        - 배열명은 상수이나 포인터는 변수
        - 따라서 배열명은 값을 바꿀 수 없지만 포인터는 바꿀수있음
        - 즉, 매커니즘은 동일하나 서로 동일한 정보를 가진 것은 아닌 것
    - 뺄셈 관계 연산
        - 포인터 간의 뺄셈, 관계 연산 가능
        - 단, 뺄셈의 경우 자료형 크기로 나눠짐
    - 동적 할당
        - 프로그램 실행 중 메모리르 할당해야 하는 경우
        - malloc 함수와 free 함수를 사용(stdlib.h 헤더 파일에 함수 원형 존재)
        - int *ptr = (int *)malloc(sizeof(int));
