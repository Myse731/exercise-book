- **변수**
    - 개요
        - 프로그래밍에서 값을 저장 하는 공간
        - 프로그램 작성시 변수에 필요한 값을 저장하고, 변수에서 가져와 사용
        - 변수에는 문자, 숫자 등 ..모든 값 저장 가능
        - 대입 연산자를 통해 오른쪽의 값을 왼쪽에 저장 가능
        - **변수 정의**: 파이선은 변수를 생성할 때 자료형을 지정하지 않아도 저장되는 값에 따라 자동으로 부여됨.
    - 변수명 작성 방법
        1. 첫 문자는 한글, 영문자 또는 ‘_’ 로 시작
        2. 나머지도 한글, 영문자, ‘_’ 여야함
        3. 변수명은 대소문자를 구문함(x1 ≠ X1)
        4. if, while, else 같은 예약어는 사용 불가
        5. 공백 하이픈 사용 불가
        - 작성 케이스
            - 스네이크 케이스(apple_cake)
            - 캐멀 케이스(AppleCake)
    - 변수와 연산
        - 변수 안에 저장된값을 더하거나 빼서 다시 변우세 저장 할 수 있음
        - 변수에 값을 대입할때는 = 대입 연산자 활용
- **자료형**
    - 프로그래밍에서 변수를 저장하는 형태
    - 숫자형 자료형, 문자형 자료형, 불(bool) 형
    - 리스트 자료형, 튜플 자료형, 딕셔너리 자료형
    - 프로그래밍 언어별 차이점은 있지만 대부분 유사함
- **자료형(숫자)**
    - 숫자 형태로 이루어진 자료형
    - 정수(10진수, 8진수, 2진수) : 소수점이 없는 숫자
    - 실수(10진수, 8진수, 2진수) : 소수점이 있는 숫자
    - 특징 : 연산 가능
- **이진수, 십진수**
    - 십진수(사람이 사용) : 0~9까지의 10개 숫자를 사용. 10이 되면 자릿수가 올라감
    - 이진수(컴퓨터가 사용) : 0, 1 두개 숫자를 사용. 2 되면 자릿수가 올라감
    - 이진수 → 십진수 변환
        - 차례대로 곱하여 더하기
    - 십진수 → 이진수 변환
        - 나누어 몫과 나머지를 쓰기
    
- 연산자
    - 산술 연산자
        - + : 더하기
        - - : 뺴기
        - * : 곱하기
        - / : 나누기
        - // : 몫 연산자
        - % : 나머지 연산자
    - 대입 연산자
        - = : 변수를 대입할때 사용
    - 산술 복합 연산자
        - += : 자기 자신을 오른쪽의 값과 더하여 다시 자신에게 대입
        - -= : 자기 자신을 오른쪽의 값과 빼서 다시 자신에게 대입
        - *= : 자기 자신을 오른쪽의 값과 곱하여 다시 자신에게 대입
        - //= : 자기 자신을 오른쪽의 값과 나누어 다시 자신에게 대입(정수형)
    - 비교(조건) 연산자
        - == : 좌항과 우항의 값이 같다
        - ! = : 좌항과 우항의 값이 다르다
        - >, < : 좌항 또는 우항의 값이 크거나 작다
        - < =, > = : 좌항 또는 우항의 값이 크거나 같고 작거나 같다.
    - 논리 연산자
        - 주어진 논리식(비교 연산자 덩어리)를 비교하여 최종 True, False 판단
        - and : 좌항 우항이 모두 True일 경우 True
        - or : 좌항과 우항 중 하나라도 True일 경우 True
        - not : True은 False로, False는 True로 반전
    - 비트 연산자
        - 십진수를 이진수 단위로 변환후, 각 자리별로 연산을 수행
        - 1은 True, 0은 False

- 문자열 자료형
    - 개요
        - 문자열 : 문자, 단어로 구성된 문자의 집합
        - 따옴표(큰따옴표, 작은따옴표 상관 없음)로 둘러싸여져 있으면 문자열(여는 따옴표, 닫는 따옴표가 일치해야 함)
        - 큰 따옴표로 둘러싸기(작은 따옴표로 둘러싸기)
    - 문자열 인덱싱
        - 문자열의 특정 위치에 있는 값을 가져올때 쓰는 방식
        - 인덱스 : 문자열 내에서의 위치, 첫번째 문자를 0번으로 취급
        - 값 가져오기 : 문자열[인덱스 번호]로 가져올수있음
    - 문자열 슬라이싱
        - 문자열의 특정 위치에 있는 값을 여러개 가져올때 쓰는 방식
        - 값 가져오기 : 문자열[시작 번호 : 마지막 번호 : 간격], 이때 번호는 가져오지 않고 그 전 번호까지만 가져옴
        - 시작 번호 생략 : 첫 글자부터 가져옴
        - 마지막 번호 생략 : 마지막 글자까지 가져옴
        - 둘 다 생략 : 전부 다 가져옴
    - 문자열과 연산자
        - 숫자형에 활용되는 연산자 +, *의 경우 문자열에는 다르게 적용됨
        - +의 경우, 두 문자열을 연결하는 연산자를 의미
        - *의 경우, 문자열을 * 뒤 횟수만큼 반복하는 연산자를 의미
    - 문자열 활용 사용되는 함수들
        - len
            - 문자열의 길이 반환
            
            ```jsx
            text = "Python! "
            length = len(text)
            
            print(length)
            ```
            
        - strip
            - 앞 뒤 공백 제거
            
            ```jsx
            user_input = "   python123   "
            clean_id = user_input.strip()
            
            print(user_input)
            print(clean_id)
            ```
            
        - split
            - 문자열 나누기
            - 특정 기준을 바탕으로 문자열을 잘라 리스트로 생성
            
            ```jsx
            hobby = "축구,농구,독서,영화"
            hobby_list = hobby.split(",") # 쉼표를 기준으로 나눔
            
            print(hobby_list)
            ```
            
        - replace
            - 특정 문자를 찾아 다른 문자로 교체
            
            ```jsx
            text = "I love Java"
            new_text = text.replace("Java", "Python")
            ```
            
        - upper, lower
            - 영문자를 대문자나 소문자로 통일
            
            ```jsx
            email = "Python_Master@Gmail.com"
            
            upper_email = email.upper()
            
            lower_email = email.lower()
            ```
            
        - count, find
            - 문자열 안에 특정 문자가 몇 개 있는지, 혹은 어디(인덱스)에 있는지 확인
            
            ```jsx
            msg = "banana"
            
            print(msg.count('a'))
            
            print(msg.find('n')
            ```
            
    
- 문자열 포매팅
    - 문자열 포매팅(Formatting) - 최신 버전 : f-string 포매팅
    - 데이터를 담을 빈 틀을 만들어두고, 나중에 원하는 값을 예쁘게 끼워넣는 방식
    - 문자열 앞에 f 를 붙여 사용
    - 변수 값을 변경하여 활용 가능
    
    ```jsx
    # f 문자열 포매팅 (1) : 문자열 앞에 f를 붙인 후, 중괄호를 열어 들어갈 값 명시
    
    name = '박강원'
    school = '부산소마고'
    
    a = f'제 이름은 {name} 입니다. 학교는 {school} 입니다.'
    
    print(a)
    
    # f 문자열 포매팅 (2) : 문자열에 들어갈 변수의 값을 참조 가능!!
    
    age = 17
    
    b = f'저는 올해 {age} 살이고, 내년에 {age + 1} 살이 됩니다.'
    
    print(b)
    ```

- 입력과 출력
    - **입력 함수 : input()**
    - 사용자(사람)으로부터 값을 입력받는 함수
    - 키보드로 우리가 원하는 값을 직접 입력하여 저장할 수 있음
    - 입력한 값을 변수에 저장하여 활용할 수 있음
    - (중요) 입력받은 값은 기본적으로 **문자열 자료형**
    - 숫자로 값을 저장하고 싶을 경우, 형 변환(자료형을 바꾸는 과정)이 필요
    
    **참고) 형 변환 : 자료형을 바꾸는 것**
    
    - 숫자(정수)형으로 바꾸고 싶을 경우 : int() 사용
    - 숫자(실수)형으로 바꾸고 싶을 경우 : float() 사용
    - 문자열 자료형으로 바꾸고 싶을 경우 : str() 사용
    - print() 추가
        - 여러 개를 출력 : , 으로 구분하여 출력
        - 줄바꿈의 경우 : print() 연달아 사용
        - 공백 기준 문자(기호) 출력 : sep
        - 줄바꿈을 제거하여 출력 : end
        
        ```jsx
        # 여러개 출력
        a = 10
        b = 20
        
        print(a, b)
        
        # 줄바꿈의 경우
        print('Hello')
        print('Python')
        
        # 공백 기준 문자 출력
        print('2026','busan','software','mister','highschool', sep = '!')
        
        # 줄바꿈 제거
        print('hi', end ='')
        print('minse')
        ```
        
- 선택 구조
    - 참, 거짓 여부를 판단후, 선택적으로 일을 처리하는 구조
    - **조건문이 참(True)면 if 아래 블록을 수행하고, 거짓(False)이면 else 아래 블록을 수행함**
    - **블록의 구분 기준 : 같은 너비의 들여쓰기**
    - **if문을 만들 때, 반드시 if 조건문 아래는 같은 너비의 들여쓰기를 해야 함. 그렇지 않을 경우 오류가 발생함.**
    
    ```jsx
    #if, else, elif 구문
    
    if 조건식 :
      수행할 문장 1
      수행할 문장 2
      
    elif 조건식 :
      수행할 문장 1
      수행할 문장 2
      
    else :
      수행할 문장 1
      수행할 문장 2
    ```
    
- 반복 구조
    - 같은 문장을 여러번 실행할때 쓰는 문법
    - 특정 조건이 참일 동안 또는 특정 횟수만큼 반복하는 문법
    - for 문과 while 문을 활용함
    - while문
        - while 명령 뒤에 오는 **조건문이 참일동안 내부 코드 반복 실행**
        - 수행할 문장이 모두 끝나면, 다시 조건문을 검사함
        - 조건문이 거짓이면 반복문 종료
        - 조건문에 변수 사용 가능, 해당변수는 반복문안에서 값변경 가능
        - while 제어 : continue, break
            - while문을 강제로 빠져나가는 문법
            - break : 현재 진행중인 반복문을 탈출할 때 사용하는 문법
            - continue : 현재 진행중인 반복문을 생략하고 다음 반복으로 넘어가는 문법(반복문 탈출이 아님!)
            
    - for 문
        - 리스트에 포함된 원소의 개수만큼 for문 내부 문장을 반복하여 수행
        - for문에 활용되는 변수 : 각 리스트, 문자열의 값을 하나씩, 순서대로 저장하는 역할
        - 더 이상 변수에 대입할 값이 없으면 반복문 종료
        - for 와 range
            - range(초기값, 최종값, 증가값) : 숫자 리스트를 자동으로 만들어주는 함수
            - 초기값 ~ 최종값 -1(더 정확히, 최종값 전 값) 까지의 숫자 리스트 자동 생성
            - 초기값은 생략할 수 있으며, 생략 시 0으로 자동 세팅
            - 증가값은 생략할 수 있으며, 생략 시 1으로 자동 세팅
        - list와 입력
            - 문자열 -> 리스트 변환에는 split(), map() 등을 사용함
            - 문자열.split() : 지정한 글자를 기준으로 문자열을 나누어, 리스트 형식으로 저장함
            - 괄호 안에 아무 내용도 명시하지 않으면, 공백을 기준으로 문자열을 나눔
            - 입력값이 숫자일 경우 -> split()에 더해 map()함수가 더 필요함
            - **map(함수, 리스트) : 리스트의 각 값을 함수로 처리하는 함수**
            - (중요) map으로 처리한 결과는 일회성이므로, 이를 저장하기 위해서는 list()로 감싸야 함
            
            ```c
            num = list(map(int, input().split()))
            
            print(num)
            
            a, b, c = map(int, input().split())
            
            a = '부산 소프트웨어 마이스터 고등학교'
            b = a.split()
            ```
            
        - list comprehension
            - 리스트 안에 for 문을 포함하는 문법 형태
            - 보다 더욱 편리하게 리스트를 만들어 낼 수 있음
            - [**표현식 for 항목 in 리스트 if 조건**] : 조건에 해당하는 리스트만 선별(필터링)
        - 2차원 리스트
            - 리스트 안에 리스트가 존재한다
            - list comprehension을 통해 간단하게 구현 가능
- 딕셔너리
    - 개요
        - 리스트 : 인덱스를 통해 리스트 값에 접근 → 내가 원하는 값의 인덱스를 알고 있어야함
        - 딕셔너리 : key를 통해 값에 접근 → 내가 원하는 값의 key만 알고 있으면 됨
        - 딕셔너리는 인덱스 대신에 key를 활용하는 리스트 라고 간단히 알고 있으면 됨
        - 기본 구조 : {key 1 : value1, key2 : value2,…} 로 key 와 value 한 쌍의 모음이 여러개 있고 콤마로 구분됨
    - 연산(1)
        - 딕셔너리 추가 : 딕셔너리 이름[key 값] = value 값
        - 참고 : key 값이 중복될 경우 최신 값으로 반영
        - 참고 : key 값으로는 리스트는 올수 없음(key 값은 불변이어야 함)
        - 딕셔너리 삭제 : del 딕셔너리 이름[key 값]

    - 상속(중요)
        
        상속(Inheritance)이란 기존에 이미 정의되어 있는 클래스의 모든 속성(변수)과 메서드를 새로운 클래스가 물려받아 그대로 사용하는 기능을 말합니다.
        
        - **부모 클래스 (Parent / Super / Base Class):** 물려주는 기존의 클래스
        - **자식 클래스 (Child / Sub / Derived Class):** 물려받는 새로운 클래스
        
        ### 왜 상속을 사용할까요? (주요 장점)
        
        1. **코드의 재사용성 향상:** 공통된 특징을 부모 클래스에 한 번만 정의해 두면, 자식 클래스에서 똑같은 코드를 다시 작성할 필요가 없습니다.
        2. **유지보수성 증대:** 공통 기능에 수정 사항이 생기면 부모 클래스만 수정하면 되므로 관리가 용이해집니다.
        3. **확장성:** 부모의 기능을 바탕으로 자식 클래스에서 새로운 속성이나 메서드를 자유롭게 추가(확장)할 수 있습니다.
