# 스택 개념 정리
* 스택은 자료구조 중 선형 자료구조로 분류된다.
* top은 최상단 위치를 가리키는 변수이고, 이를 통해 값이 삽입, 삭제 된다.(한 쪽에서만 삽입, 삭제가 이루어짐)
* 마지막에 들어간 값이 처음으로 빠져나오는 구조(Last In First Out, LIFO 구조)
* top의 초기값은 -1이다.
* 삽입과 삭제는 모두 최상단에서만 이루어진다.


#### Push(값) : 값 삽입
1. 오버플로우 확인( 오버플로우라면 오류발생)
2. top의 값을 1 증가시킴
3. top을 1 증가시킨 곳에 값 삽입


### Pop() : 삭제
1. 언더플로우 확인( 언더플로우라면 오류발생)
2. 삭제할 값을 임의의 변수에 할당
3. 현재 top 위치에 있는 값을 None으로 만든다.
4. top의 값을 1 감소시킨다
5. 이후 임의의 변수에 저장 되어있는 삭제 될 값을 반환한다.


### Peek() : 최상단 값 확인
1. 언더플로우 확인한다.(언더플로우라면 값이 반환되지 않는다.)
2. 최상단의 값을 반환한다.



### Overflow : 오버플로우
* 오버플로우란 스택의 값이 꽉차있을때 삽입을 실행하면 더이상 값을 삽입할수없을때 일어나게 되는 오류이다.
* 꽉차있는지 확인하는 방법은 top >= len(stack)-1 이다.
   

### Underflow : 언더플로우
* 언더플로우란 스택의 값이 비어있을때 삭제를 실행하게 되면 삭제할 값이 존재하지 않아서 일어나는 오류이다.
* 비었는지 확인하는 방법은 top == -1 이다
