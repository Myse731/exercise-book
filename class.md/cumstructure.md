# 데이터 표현의 기초

## 컴퓨터는 왜 0과 1만 쓸까?

#### 컴퓨터는 전기로 작동하는 기계로 복잡한 10진수를 사용하는 것보다 전압이 높음과 낮음 두 상태만 구분하는 것이 훨씬 정확하고 빠르다

## 데이터 단위

## 비트

- 0 과 1을 표현하는 가장 작은 단위
- n비트로 표현할 수 있는 정보의 가짓수는 2^n
- 1비트가 추가될수록 표현 가능한 정보는 2배씩 늘어난다

## 단위

```python
1. Bit(비트)

* 컴퓨터가 처리하는 가장 작은 단위
* 0 또는 1 두가지 상태만 가질수 있다

2. Byte(바이트)

* 비트 8개를 하나로 묶은 단위
* 2^8 = 256가지의 정보를 표현할 수 있어, 영문자 한 글자를 저장할수있는 기본 단위가 됩니다.
```

```python
컴퓨터는 10진수 1000 가장 가까운 2진수인 2^10(=1,024)을 기준으로 단위를 올린다

1 Byte
| x 1,000
KB(KiloByte) : 2^10 Byte
| x 1,000
MB(MegaByte) : 2^10 KB
| x 1,000
GB(GigaByte) : 2^10 MB
| x 1,000
TB(TeraByte) : 2^10 GB

---- 1000 이랑 1024랑 같다고 보는게 괜찮을까?----

용량이 줄어든 것은 아니고, 숫자를 세는 기준(단위)이 달라서 생기는 현상
최신 표준은 혼란을 막기 위해 1,024 단위를 KiB, MiB, GiB(키비, 메비, 기비)라고 부르기로 함.

단, 정보처리기능사 같은 기초 자격증 시험에는 여전히 1,024B = 1KB로 봄.
```

## 16진수

- 한 자리에 0~15까지의 숫자로 수를 표현
- 2진수는 너무 길고 10진수는 컴퓨터와 친하지 않기 때문에 개발자는 16진수를 활용
- 활용 예시 : 웹 디자인, 메모리 주소, 네트워크, 임베디드 레지스터 제어
- 인간의 인지 오류 감소, 메모리 구조의 직관적 파악, 하드웨어 상태 시각화, 문서화 및 통신 효율성

### 사용이유 : 데이터 가독성과 압축률

- 32비트 데이터를 표현할 때, 2진수는 32개 숫자가 필요하지만 16진수는 단 8개면 충분하다.
4:1의 압축비가 엔지니어들이 16진수를 사용하는 실질적인 이유이다.

### 2진수 → 16진수

```python
1. 4자리 나누기

1101 | 0110 #뒤에서부터 끊어줌

2. 10진수 계산

1101 = 13

0110 = 6

3. 16진수 매핑
13 -> D
6 -> 6
'D6'
```

### 16진수 → 2진수

## 이진수의 덧셈과 뺄셈

- 덧셈
    
    ```css
    1001 + 1101
    
    = 10111
    
    1100 + 10110
    
    = 100010
    ```
    
- 뺄셈
    
    ```css
    기본 규칙
    덧셈기와 뺄셈기를 모두 만들면 컴퓨터 칩이 크고 복잡해짐. 보수를 이용하면
    기존의 덧셈기만으로 뺄셈을 완벽하게 수행할 수 있어 효율적
    '부족한 수를 채워주는 수'라는 뜻
    ex) 3에 대한 10의 보수 : 7, 4에 대한 10의 보수 : 6
    수학적으로 뺄셈을 덧셈으로 바꾸는 도구이다. ex) 15 - 3
    1. 10에 대한 3의 보수 : 7
    2. 15+7 = 22
    3. 올림수 10을 빼고 남은수 : 12
    
    2진수에서의 보수
    • 1의 보수 : 모든 비트를 반대로 뒤집은 수
     (1010 -> 0101) -> "0과 1을 서로 보충해주는 관계)
     
    2의 보수 : 1의 보수에 1을 더한 수 -> "컴퓨터가 사용하는 진짜 음수"
    ```
    

## 부호의 표현과 데이터 한계

- 4비트 부호 범위 : -8 ~ 7
- 5비트 부호 범위 : -16 ~ 15
- n비트 부호 범위 : -2^n ~ 2n^n-1 -1

```css
만약 최대 7(0111)인 값에 1(0001)을 더하면 어떻게 될까?

= 1000(8)

양수 + 양수 = 음수

양수 + 양수를 했는데 음수가 되거나 음수 + 음수를 했는데 양수가 나오는 경우
즉, 계산 겨로가가 부호비트 칸을 침범하는 현상을 오버(언더 플로우)라고 한다.
```
